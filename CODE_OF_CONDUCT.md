# CAGE-lite Code of Conduct

## Our commitment

CAGE-lite is intended to support open technical discussion, research, implementation, and evaluation of assurance controls for AI-agent systems.

Everyone participating in the project is expected to communicate professionally and respectfully, including when they disagree about architecture, terminology, implementation choices, or project claims.

## Expected behavior

Participants should:

- communicate clearly and respectfully
- critique ideas and technical work rather than individuals
- provide context and evidence for technical claims
- accept constructive feedback
- acknowledge mistakes and correct them
- respect different levels of experience and professional backgrounds
- keep discussions relevant to the project
- avoid presenting speculation as established fact

## Unacceptable behavior

The following behavior is not acceptable:

- harassment, threats, intimidation, or personal attacks
- discriminatory or demeaning language
- unwanted sexual attention or inappropriate content
- deliberate disruption of project discussions
- publishing another person's private information without permission
- repeated bad-faith arguments after maintainers have clarified a decision
- impersonation, misrepresentation, or knowingly false claims about contributors or the project
- retaliation against someone who reports a concern

## Technical disagreement

Strong technical disagreement is allowed and can improve the project.

Participants may challenge the CAGE framework, implementation, architecture, terminology, evidence, or positioning. Criticism should remain specific, factual, and directed at the work.

Technical disagreement does not justify hostile or abusive behavior.

## Scope

This Code of Conduct applies to:

- repository issues and pull requests
- code reviews and project discussions
- project documentation
- project-related public or private communication
- events or forums where someone is representing CAGE-lite

## Reporting concerns

Conduct concerns should be reported privately to the project maintainer using the contact information available through the maintainer's GitHub profile.

Do not open a public issue when a report contains sensitive or personal information.

Reports will be reviewed as promptly and impartially as practical. Information will be shared only with people needed to evaluate and address the concern.

## Enforcement

Project maintainers may take actions including:

- requesting a correction or apology
- editing or removing inappropriate content
- closing or locking a discussion
- rejecting a contribution
- issuing a temporary participation restriction
- permanently banning a participant from project spaces

The response will depend on the seriousness, persistence, and impact of the behavior.

## Good-faith participation

This Code of Conduct is intended to protect productive participation. It is not intended to prevent legitimate technical criticism or respectful disagreement.

Everyone participating in CAGE-lite is expected to follow it.
